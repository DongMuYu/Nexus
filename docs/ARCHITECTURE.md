# Nexus 物理引擎架构设计

## 概述
Nexus 是一个实时物理引擎，专注于粒子系统和刚体动力学模拟。

## 核心架构

### 1. 核心系统 (Core)
- **Application**: 主应用类，负责初始化、主循环和资源管理
- **Config**: 配置管理系统，支持运行时配置热重载
- **Profiler**: 性能分析器，提供详细的性能指标
- **ThreadPool**: 线程池管理，优化多线程任务调度

### 2. 物理引擎核心 (Physics)
- **World**: 物理世界管理，包含实体管理和场景组织
- **Solver**: 物理求解器，实现多种数值积分方法
- **SpatialHash**: 空间哈希/四叉树空间分区系统
- **Materials**: 材料属性系统，定义物理材质参数
- **Constraints**: 约束系统，支持各种物理约束

### 3. 粒子系统 (Particles)
- **Particle**: 基础粒子类，包含物理属性和状态
- **ParticleSystem**: 粒子管理系统，负责粒子的创建、更新和销毁
- **Emitters**: 粒子发射器，支持点和区域发射模式

### 4. 渲染系统 (Rendering)
- **Renderer**: 主渲染器，基于SFML的渲染管线
- **DebugDraw**: 调试绘制系统，可视化物理调试信息
- **PostProcess**: 后处理效果系统

### 5. 用户界面 (UI)
- **UIManager**: UI管理系统，基于Dear ImGui
- **Controls**: UI控件系统，包含属性面板、工具栏、统计覆盖

### 6. 工具类 (Utils)
- **Math**: 数学库，包含向量、矩阵、随机数等
- **Logger**: 日志系统，支持多级别日志输出
- **FileIO**: 文件读写系统，支持配置和资源加载
- **Timer**: 高精度计时器，用于性能测量

## 数据流
1. 应用初始化 → 加载配置 → 创建物理世界
2. 主循环: 输入处理 → 物理模拟 → 渲染 → UI更新
3. 物理模拟: 空间分区 → 碰撞检测 → 约束求解 → 状态更新
4. 渲染: 粒子渲染 → 调试绘制 → 后处理 → UI渲染

## 性能优化
- 多线程物理模拟
- 空间分区优化碰撞检测
- 批处理渲染
- 内存池管理粒子对象
